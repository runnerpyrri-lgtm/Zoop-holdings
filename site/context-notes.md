# 로봄 웹 허브 컨텍스트

## 제품 구조

- 로봄은 세 알림 앱의 소비자용 공용 입구다.
- 현재 실서비스는 야외봄, 청약봄, 러닝봄이며 기존 공개 URL을 마이그레이션 기간 동안 유지한다.
- 홈페이지는 로봄 본사 저장소의 `site/`에서 운영한다.

## 브랜드 표현 원칙

- 로봄의 핵심 문장은 `중요한 순간을 먼저 봅니다.`다.
- 세 앱은 정보 포털이 아니라 각자 `행동해야 할 타이밍`을 알려주는 서비스로 설명한다.
- 상표 등록 여부가 확정되지 않았으므로 등록·출원 상태를 사이트에 단정해서 표기하지 않는다.
- 캐릭터는 사용하지 않는다. 야외의 공기, 열리는 달력, 출발 트랙처럼 앱이 알려주는 순간을 추상 그래픽으로 표현한다.
- 따뜻함은 인물이나 마스코트가 아니라 크림색 종이 질감, 생활어 문장, 부드러운 색 대비로 만든다.

## 모바일 앱 셸

- 폰(780px 이하)에서는 첫 화면이 런처형 앱 홈으로 분기된다. 데스크톱 편집형 레이아웃은 그대로 둔다.
- 앱 홈 타일에는 `18:20`, `D-1` 같은 실시간 수치를 표기하지 않는다. 수치는 각 앱에 들어가 확인하게 하고, 홈에는 생활어 문구만 담는다.
- 하단 탭바는 `env(safe-area-inset-bottom)`을 존중하고 모든 터치 영역은 44px 이상을 유지한다.

## 배포 경로

- 정본 배포: GitHub Pages 자동배포(`deploy-site-pages.yml`) — main 머지 = 배포. 미리보기 URL: https://robom-labs.github.io/robom/
- robom.kr 도메인은 DNS 전환 전까지 기존 Sites 배포본을 서빙한다. 전환 절차는 checklist 참조.

## 현재 직접 연결 URL

- 야외봄: https://runningcall.vercel.app
- 청약봄: https://robom-labs.github.io/homebom/
- 러닝봄: https://robom-labs.github.io/runningbom/

## 출시 전 위험

- 일부 모바일·인앱 브라우저는 `target="_blank"` 새 창을 차단할 수 있으므로 세 앱 진입 링크는 같은 창에서 이동한다.
- 모바일 상단 앱 링크는 실제 손가락 터치를 위해 최소 44px 높이를 유지한다.
- 러닝봄의 원본 대회 데이터에는 외부 `http://` 링크가 남아 있다. 로봄 허브는 해당 앱의 입구까지만 HTTPS로 연결하고 외부 대회 링크를 직접 중계하지 않는다.
- 도메인 구매, DNS 변경, 결제, 상표 출원은 회장 계정의 승인·결제가 필요한 외부 상태 변경이다.
