대화 로그는 비공개 GitHub `robom-labs/ai-meeting-logs`의 표에 적힌 회의실 폴더에 둔다. 기존 로컬 `Robom-AI-Logs` 경로는 사용하지 않는다.

모든 회의실은 `COMPANY-MODE.md`를 공통으로 읽고, 비단순 요청에서 관련 제품팀·전문팀·반대검토 역할을 병렬 또는 웨이브로 소집한다.

## 사용자가 말하는 방법

- 평소처럼 요청해도 복잡한 작업에는 회사 모드가 자동 적용된다. 요청이 단순한지는 AI가 판단한다.
- 여러 팀의 총동원을 확실히 원하면 문장 앞에 `회사모드로`를 붙인다. `총동원`, `회의해서`, `논스톱`도 같은 뜻으로 처리한다.
- 검토만 원하면 `회사모드로 점검하고 제안만 해줘`라고 말한다.
- 실제 수정과 검증까지 원하면 `회사모드로 점검한 뒤 필요한 수정과 검증까지 해줘`라고 말한다.
- 01~03 총괄팀에서 `논스톱` 또는 실행형 요청은 일반 코드 변경의 병합·기존 파이프라인 배포·운영 검증까지 상시 승인한다. 결제, 계정·권한, 비밀값, 대량 삭제, 복구 불가능한 변경, 법적 동의는 자동 승인되지 않는다.

## 회의실별 요청 예시

- 아래 예시는 방의 권한을 제한하지 않는다. 01~03 어디에서나 모든 제품·사이트·공통 운영을 요청할 수 있다.
- 01은 `회사모드로 세 앱의 현재 상태를 비교하고 이번 주 출시 우선순위를 정해줘`라고 요청한다.
- 02는 `회사모드로 홈페이지를 모바일 사용자 관점에서 점검하고 개선안과 필요한 수정까지 진행해줘`라고 요청한다.
- 03은 `회사모드로 야외봄 위치와 날씨 기능을 점검하고 문제를 수정한 뒤 모바일에서 검증해줘`라고 요청한다.
- 04는 `회사모드로 청약 알림 누락 위험을 확인하고 필요한 수정까지 해줘`라고 요청한다.
- 05는 `회사모드로 러닝 대회 접수 화면을 점검하고 모바일에서 누르기 쉽게 개선해줘`라고 요청한다.

각 새 채팅에는 해당 회의실의 `PROMPT-CODEX.md` 또는 `PROMPT-CLAUDE.md` 전체를 붙여넣는다.
