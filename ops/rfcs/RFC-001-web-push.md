# RFC-001 — 서버 Web Push (앱 꺼져 있어도 알림)

상태: **제안(회장 승인 대기)** · 분류: MAJOR (구조·백엔드·비용) · 작성: 본부장

## 왜 필요한가
회사의 약속은 "놓치면 끝나는 순간을 제때 알려준다"이다. 그런데 현재 세 앱 모두
**앱 화면이 열려 있을 때만** 알림이 울린다(각 앱에 정직하게 고지되어 있음).
핸드폰을 주머니에 넣고 있어도 울리게 하려면 서버에서 보내는 푸시(Web Push)가 필요하다.

## 제안: 한 번에 3개 앱 말고, 줍줍콜 하나로 먼저 검증
(외부 자문 검토 의견 반영 — 공통 플랫폼부터 만들면 실패 비용이 3배가 된다)

1. **1단계 (검증)**: 줍줍콜에만 Web Push 도입
   - 필요한 것: VAPID 키 1쌍, 구독 저장소(Supabase 테이블 1개), 발송 스케줄러
     (Supabase Scheduled Function 또는 GitHub Actions cron), 서비스워커 push 핸들러
   - 줍줍콜이 이미 Supabase Edge Function을 쓰고 있어 인프라 추가가 가장 적다
   - 성공 기준: 앱을 완전히 닫은 안드로이드 폰에서 접수 시작 알림 수신
2. **2단계 (확장)**: 검증되면 같은 구조를 PushRun → 러닝콜 순으로 복제
3. **하지 않는 것**: 지금 코드에 실험적 API(Notification Triggers 등)를 붙이는 것 —
   과거에 제거한 이유가 있다(비표준·미지원).

## 비용·제약 (승인 전 확인 필요)
- Supabase 무료 티어로 시작 가능(구독 수천 건 수준). 유료 전환 조건을 문서화한다.
- iOS는 홈 화면에 추가(PWA 설치)한 경우에만 Web Push 수신 가능(iOS 16.4+) — 마케팅 문구 주의.
- 개인정보: 구독 토큰 저장이 시작됨 → 개인정보처리방침 페이지가 선행 조건.

## 착수 조건
- 회장 승인 + 1단계 상세 설계서(테이블 스키마, 발송 시나리오, 실패 재시도) 검토 통과.
- 이 RFC 승인 전에는 어떤 앱에도 푸시 코드를 넣지 않는다.
