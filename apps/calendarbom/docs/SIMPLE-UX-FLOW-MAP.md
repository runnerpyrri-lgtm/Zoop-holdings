# 단순 UX 흐름 지도 (V6 → V7 갱신)

> V7 변경: 최근이 있으면 첫 화면은 [지난번과 같게]+최근 2+[약 먹기][병원]+[다른 일정] = 큰 선택 6개 이하.
> 기념일: [나] 선택 가능, 기준일 [바꾸기] 제공, 관계별 추천 4개, 같은 날짜는 자동 병합.
> 약: 기간(계속/1·2주일/한 달/이번만) 요약 행, 라벨은 시간에 맞춰 자동 제안. 미래 복용은 "예정".
> 상세: [수정]으로 날짜·제목·시간·장소·메모 편집. 저장 실패 시 시트가 닫히지 않는다.

날짜 탭 이후 모든 흐름. `[ ]`=칩/버튼, `✓`=접힌 요약(변경 가능), `(자동)`=canInfer 기본값.

## 공통 시트 첫 화면

```
7월 18일에 어떤 일정을 추가할까요?

[지난번과 같게: 엄마 병원]        ← 있을 때만
최근 사용: [엄마 병원] [내 약]     ← 있을 때만, 최대 2

[약 먹기] [병원]
[약속]   [생일·기념일]
[할 일]  [직접 입력]
```

## 일반·할 일·직접 입력 (결정 2)

```
①내용 → ✓일정 ②시간(빠른 칩 4 or 시간 바꾸기) → 요약 → [저장]
(자동) 알림: 정각 / 대상: 나 / 반복: 이번만 — 요약에 표시, [변경] 가능
```

## 병원 (결정 2~3)

```
①[병원] → ✓ (자동)누가: 나 [바꾸기] ②시간 → 요약 → [저장]
저장 직후 toast: [다음 병원 일정 만들기]
```

## 약속 (결정 3)

```
①[약속] → ②누구와? [사람칩...] [사람 추가] → ③시간 → 요약 → [저장]
```

## 약 먹기 (첫 등록 결정 3~4)

```
①[약 먹기] → (자동)누가: 나 [바꾸기]
②하루 몇 번? [1번] [2번] [3번]
③시간 확인: 아침 오전 8시 [변경] / 저녁 오후 7시 [변경]   ← 채워져 있음
(자동) 반복: 매일 [바꾸기] / 이름: 아침약·저녁약 [직접 입력]
→ 요약("시간은 처방받은 안내에 맞게 바꿔주세요.") → [저장]
```

## 생일·기념일 (첫 등록 결정 4)

```
①[생일·기념일] → ②누구의 날? [사람칩...] [사람 추가]
③어떤 날? [생일] [결혼기념일] [사귀기 시작한 날] [기억하는 날]
   └ 사귀기 시작한 날 → [100일] [200일] [1주년] [다른 날→300/500/1000/N일/N개월/N주년]
   (자동) 기준일 = 달력에서 선택한 날짜 [변경]
   계산 확인 문장: "4월 10일을 1일째로 계산해요. 100일은 7월 18일이에요."
④알림: [7일 전] [하루 전] [당일 오전]
→ 요약 → [저장]  ← 선택(체크)한 날만 생성
```

## 일정 상세 (기존 일정 탭)

```
제목 · 자연어 일시 · 상태
[완료] [시간 바꾸기] [복사]
[다음 일정 만들기]           ← 병원 등
[공유] [삭제]
반복 일정의 시간/삭제 → 범위 선택: [오늘 일정만] [오늘부터 앞으로] [반복 전체]
```

## 약 확인 (알림 탭·날짜 상세)

```
오전 8시  엄마 혈압약   [먹었어요] [조금 있다가▾5/10/30/60분] [건너뛰었어요]
```

## 공유 (한 건)

```
[공유] → 미리보기(제목·일시·장소만, 메모 제외 기본)
"약 이름이 들어 있어요. 보낼 사람을 확인해 주세요."  ← 약 일정일 때
[문자로 보내기] [달력 파일(ICS)] — 다른 일정·전체 달력은 절대 포함 안 됨
```

## 복구 규칙

- 시트 닫힘/뒤로 → draft 자동 보존, 같은 날짜 재진입 시 복원 제안.
- 저장·삭제 직후 toast [되돌리기] 8초.
