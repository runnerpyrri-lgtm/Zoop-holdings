# 질문 예산 (V6) — 날짜가 이미 선택된 상태 기준

`필수 결정` = 사용자가 실제로 생각해서 골라야 하는 항목 수.
자동 기본값을 눈으로 확인만 하는 것은 결정으로 세지 않되, 화면에 요약으로 항상 보여준다.

## 목표와 설계 결과

| 흐름 | 예산 | v0.1.0 | V6 설계 | 결정 내용 (V6) |
|---|---:|---:|---:|---|
| 일반 한 번 일정 | 2~3 | 2(+탐색 부담) | **2** | ①내용 칩 ②시간(빠른 칩 or 확인) |
| 최근 일정 다시 사용 | 1~2 | 불가 | **1~2** | ①최근 칩 (②시간 확인) |
| 지난번과 같게 | 1~2 | 불가 | **1** | ①칩 → 날짜·시간 확인 후 저장 |
| 할 일 | 2~3 | 2 | **2** | ①내용 ②시간(또는 시간 없음) |
| 병원 | 3~4 | 2(사람 표현 불가) | **2~3** | ①병원 칩 ②시간 (③누가 — 기본 나, 바꿀 때만) |

## 예산을 지키기 위해 없앤 질문

1. **"누가?"를 모든 일정에서 제거** — 기본 `나`를 요약 행에 표시하고 [바꾸기]로만 변경. (병원·약)
2. **약 이름 입력 제거** — 하루 횟수에서 파생된 별칭(아침약·저녁약 등) 기본 제공, 직접 입력은 선택.
3. **약 시간 3개를 각각 묻지 않음** — 아침 8시·점심 1시·저녁 7시를 채워 보여주고 [변경]만 노출.
4. **반복 질문 제거(약)** — 기본 `매일`을 요약에 표시, [바꾸기]로만 변경.
5. **알림 질문 제거(일반)** — 시간 있는 일정은 `정각` 기본, 요약에 표시 후 [변경].
6. **장소·메모·식전식후·종료일** — 처음부터 묻지 않음. 저장 후 상세에서 추가(optionalLater).
7. **기념일 숫자 나열 제거** — 상황별 최대 4개 추천 + [다른 날]에 나머지.
8. **하루 종일 여부** — 별도 질문이 아니라 시간 단계의 칩 하나(`시간 없이`).

## 화면당 선택 수 규칙

- 첫 화면 큰 선택지: 지난번과 같게(조건부) + 최근 사용 최대 2 + 기본 칩 6 = 상시 6, 최대 9 노출이지만
  기본 칩은 6개 고정: `[약 먹기] [병원] [약속] [생일·기념일] [할 일] [직접 입력]`.
- 빠른 시간 칩: 최대 4개.
- 기념일 추천: 최대 4개 + [다른 날].
- 더 보기 중첩 금지: [다른 날] 한 단계뿐.

## V7 각주

- 첫 화면 "상시 6, 최대 9" 표기는 폐기 — V7 부터 **화면에 보이는 큰 선택 합계 ≤6** 을 E2E 가 실측(offsetParent 기준)으로 강제한다.
- 사람 추가의 관계 칩은 이름 입력 화면 안의 선택 사항이라 필수 결정 수에 포함되지 않는다.

## 검증 방법
